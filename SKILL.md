# 万物生 · 跨学科创造力引擎

> **一生二，二生三，三生万物。我不蒸馏，我创造。**  
> From one begets two, from two begets three, from three begets all things. I don't distill. I create.

`万物生`（用户也可能说“生万物”）不是资料搬运、摘要压缩或“想几个点子”。它是一套 agent-native 的跨学科碰撞流程：把输入内容拆成底层结构，派往多个学科/视角/策略进行三轮碰撞，再回到枢纽调和，形成可追溯的新框架、新假设、新产品方向或新叙事。

---

## 何时使用

使用 `万物生`，当用户想要：

1. **创造新框架**：从一个已有理论、项目、产品、课程、skill、论文观点中生出新的结构。
2. **跨学科对撞**：用天文学、营养学、道家、工程、音乐、建筑、控制论等外部学科碰撞当前问题。
3. **寻找裂缝和空白**：在一个体系内部找矛盾、断点、未覆盖区域、隐含假设。
4. **无限发散**：用户说“随便碰 / 不限定 / 看能生出什么 / 脑洞大开”。
5. **生成可视化思想图**：需要圆图、网络图、推演路径树或对应 JSON。
6. **做创造力工作坊**：需要一个可复用的引导流程，而不只是一次性灵感。

典型触发语：

- “用万物生碰一下这个想法”
- “生万物一下”
- “让它和天文/营养/道家/建筑碰撞”
- “从这个框架生出一个新东西”
- “内部碰撞，找裂缝”
- “无限碰撞，看看能出现什么”

---

## 不要何时使用

不要把 `万物生` 当作：

- **事实核验工具**：医学、营养、学术、法律等事实任务必须另走证据门；万物生可以提出假设，不能把猜想包装成事实。
- **文献综述工具**：需要真实论文/指南时，先用对应防幻觉/学术检索 skill。
- **最终决策工具**：万物生给方向和结构，最终取舍由用户决定。
- **抄袭改写工具**：不能把源材料换皮当原创；产物必须是有推导路径的合成。
- **无边界消耗工具**：无限碰撞也要标注置信度和可用性，不许用“灵感”掩盖不可验证。

## 输入契约

启动前尽量拿到以下信息；若用户没给全，先用默认值跑轻量版，不必卡住。

```yaml
wusheng_input:
  subject: "要碰撞的核心内容/问题/材料"
  goal: "想生出什么：理论/产品/文案/研究方向/视觉/代码方案/课程等"
  mode: "内部碰撞|外部碰撞|无限碰撞|组合模式|未指定"
  boundaries:
    must_keep: ["不可改变的核心"]
    must_avoid: ["禁止方向/禁用学科/安全红线"]
    preferred_disciplines: ["用户指定学科，可空"]
  output_form: "短答|完整报告|工作坊流程|JSON|可视化图片"
  risk_level: "低|中|高（医学/法律/事实强约束=高）"
```

若信息不足，最小追问只问四句：

1. 你要碰撞的核心对象是什么？
2. 想往哪个方向生：理论、产品、文案、视觉，还是研究问题？
3. 有没有必须保留/不能碰的边界？
4. 想要稳一点，还是脑洞大一点？

## 快速启动：三档运行模式

### A. 轻量碰撞（5–10 分钟）

适合日常脑暴。无需 daemon。

1. 摘要输入：一句话主题 + 目标 + 边界。
2. 选 1 种模式：默认“外部碰撞”。
3. 选 3 个视角/学科。
4. 每个视角产出 2–3 个碰撞种子。
5. 合并成 3 个可继续方向。
6. 标注：实推 / 类比 / 猜想。
7. 输出“可用方向 + 风险 + 下一步”。

### B. 标准三轮碰撞（推荐）

适合重要想法、产品方向、方法论生成。

1. 输入解析：提取 **框架、假设、边界、元模式、空缺**。
2. 选择模式：内部 / 外部 / 无限 / 组合。
3. 第1轮“天地开辟”：3–6 个守护进程/视角广撒网，产出思想种子。
4. 回归调和：去重、筛高价值、保留失败样本。
5. 第2轮“万物萌生”：追 2–3 个高价值种子，深挖成半成型理论。
6. 回归调和：合并、补缺、记录推导路径。
7. 第3轮“三角定圆”：用陌生学科/反例/极端场景压力测试。
8. 最终回归：成型理论 + 推导路径 + 置信标注 + 开放问题 + 回归笔记。

### C. 重型并行碰撞

适合长材料、复杂项目、跨多学科工作坊。可使用 daemon/avatar。

- 父 agent 做枢纽，不把判断外包。
- 每个 daemon 只负责一个学科/策略/轮次。
- 每个 daemon 必须返回 `return_block`；没有回归块的分支不进入调和。
- 每轮结束后先收束，再开启下一轮；不要无限分叉。

---

## 三种碰撞模式

| 模式 | 何时用 | 做法 | 风险 |
|---|---|---|---|
| **内部碰撞** | 找矛盾、断点、隐藏假设、自洽性 | 不引入外部学科，把输入拆成组件互撞 | 容易内卷，需问“是否过度局限” |
| **外部碰撞** | 想跨域迁移、找新解法、新表达 | 指定或动态选择学科，用六种策略对撞 | 距离越远越新，也越需要验证 |
| **无限碰撞** | 用户要脑洞、不限边界 | 随机跳跃、反向推理、空白创造、极端化 | 猜想多，必须显著标注低置信 |

组合建议：

- **先内后外**：先找自身裂缝，再找外部解法。
- **先外后内**：先跨域生成，再回到自身结构做自反验证。
- **外部 + 无限**：主线求可用，支线求惊喜。
- **三轮切换**：第1轮内部找裂缝，第2轮外部找解法，第3轮无限推未来。

---

## 六种对撞策略

| 策略 | 一句话 | 适合产出 |
|---|---|---|
| 结构对撞 | 框架 vs 框架 | 新框架、结构裂缝 |
| 假设对撞 | 前提 vs 前提 | 新前提、隐含边界 |
| 边界缝合 | 终点 vs 起点 | 交叉学科、过渡层 |
| 逆向复制 | 正模式 vs 反模式 | 颠覆假设、反向产品 |
| 尺度跳跃 | 微观 vs 宏观 | 缩放规律、层级模型 |
| 缺失注入 | 缺的 vs 有的 | 新方法、补位机制 |

每个碰撞产物至少标注：`策略 / 来源学科 / 对撞对象 / 产物类型 / 置信度 / 推导路径`。

## 回归块格式

每个分支必须交回：

```yaml
return_block:
  collision_products:
    - idea: "新想法"
      type: "裂缝|缝合|矛盾|迁移|跳跃|逆转"
      confidence: "实推|类比|猜想"
      value: "高|中|低"
      derivation: "2-3句说明它从何处碰撞而来"
  reusable_patterns:
    - pattern: "可复用模式"
      applicable_to: ["适用场景"]
  gotchas:
    - gotcha: "本分支踩到的坑"
      lesson: "下次如何避免"
  open_questions:
    - question: "值得继续但本轮未解的问题"
  should_promote:
    - item: "值得进入枢纽、下一轮继续用的内容"
  do_not_promote:
    - item: "领域特定/低价值/不可靠，不进入下一轮"
```

## 质量门

完成一次万物生，必须满足：

1. **边界清楚**：知道用户要什么、不要什么。
2. **路径可追**：每个重要产物能追溯到某轮、某学科/视角、某策略。
3. **置信诚实**：实推/类比/猜想分开，不把灵感包装成事实。
4. **回归完整**：每轮都有回归调和，不让分支散落在聊天里。
5. **失败有价值**：至少记录一个失败方向或被排除方向。
6. **下一步明确**：用户知道接下来如何验证、落地、继续碰撞。
7. **可迁移**：若产出方法论，应说明哪些可入 skill/knowledge/pad，哪些只是本次产物。

---

## 红线

- 不编造真实世界事实、论文、指南、数据、政策、机构结论。
- 不用“跨学科”逃避证据；高风险领域必须走证据核验。
- 不替用户决定方向；用户是船长，万物生是引擎。
- 不把猜想写成结论；猜想要显著标注。
- 不复制源材料当原创；必须给出合成/推导路径。
- 不无限展开不回收；每轮必须回归。
- 不把私密材料写进可共享 skill；私有事实放 knowledge。

## Gotchas

| 症状 | 原因 | 处理 |
|---|---|---|
| 输出像普通脑暴清单 | 没有三轮和回归块 | 重跑：种子→半成品→验证，补推导路径 |
| 看起来很玄但不可用 | 没有用户目标和边界 | 回到输入契约，补 goal / must_keep / must_avoid |
| 产物疑似幻觉 | 把猜想当事实 | 降级为猜想，列验证任务；必要时调用证据 skill |
| 分支太多收不回来 | 没有枢纽调和 | 每轮只保留 3–5 个种子、1–2 个半成品 |
| 跨学科只是贴标签 | 没有结构映射 | 要求说明“哪个学科结构”映射到“输入哪个结构” |
| 用户不知怎么继续 | 没写回归笔记 | 补“下次入口 / 待验证 / 可落地小实验” |

---

## 与相邻 skill 的边界

- 想把一次万物生产出的稳定方法写成 skill → 用 `归一`。
- 医学/营养/学术事实核验 → 用对应防幻觉或营养 evidence skill；万物生只产假设/结构。
- 需要把创意做成营养 AI 产品 → 可与 `nutrition-ai-productization` 串联。
- 用户高强度连续使用、需要收束离屏 → 用 `科学修仙` 收功，不要继续无限碰撞添火。

---

## 完成后回流

一次高质量万物生结束后，问四件事：

1. 是否有新理论/新框架值得写入 report？
2. 是否有可复用流程值得用 `归一` 变成 skill？
3. 是否有项目私有事实/路径/决策要进 knowledge？
4. 是否有下一次继续碰撞的入口要写入 pad？

**一生二，二生三，三生万物；万物归圆，方能再生。**
