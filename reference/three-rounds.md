# 三轮碰撞详细流程

> **天地开辟，万物萌生，三角定圆。**

## 总览

```
输入内容
    │
    ▼
深度解析（提取5种底层结构）
    │
    ═════════════════════════════════
    ║  第1轮：天地开辟（广度撒网）  ║
    ═════════════════════════════════
    │
    ├──→ 守护进程A（学科1 + 策略X）
    ├──→ 守护进程B（学科2 + 策略Y）
    ├──→ 守护进程C（学科3 + 策略Z）
    └──→ 守护进程N...
          │
          ▼
    回归枢纽 → 筛选高价值种子
    │
    ═════════════════════════════════
    ║  第2轮：万物萌生（深度追踪）  ║
    ═════════════════════════════════
    │
    ├──→ 守护进程D（新学科 + 深挖策略）
    ├──→ 守护进程E（同学科 + 换角度）
    └──→ 守护进程F（跨种子缝合）
          │
          ▼
    回归枢纽 → 合并半成品 → 识别成型方向
    │
    ═════════════════════════════════
    ║  第3轮：三角定圆（反刍验证）  ║
    ═════════════════════════════════
    │
    ├──→ 守护进程G（完全不同学科攻击）
    └──→ 守护进程H（极端角度压力测试）
          │
          ▼
    回归枢纽 → 最终调和 → 输出
```

---

## 第1轮：天地开辟（广度撒网）

### 目标
尽可能多地打开方向，不做评判。

### 执行步骤

1. **解析输入**：提取5种底层结构（框架、假设、边界、元模式、空缺）
2. **匹配学科**：每个底层结构匹配1-2个学科，总共3-6个守护进程
3. **分配策略**：每个守护进程分配1种对撞策略（参见 six-strategies.md），尽量覆盖不同策略
4. **分叉外出**：所有守护进程同时出发，各自在负责的学科维度里碰撞
5. **收集回归块**：所有守护进程返回碰撞产物 + 回归块

### 守护进程分配示例

| 守护进程 | 撞什么底层结构 | 用什么学科 | 用什么策略 |
|---------|-------------|----------|----------|
| D1 | 框架 | 物理学 | 结构对撞 |
| D2 | 假设 | 道家哲学 | 假设对撞 |
| D3 | 边界 | 生物学 | 边界缝合 |
| D4 | 空缺 | 经济学 | 缺失注入 |
| D5 | 元模式 | 音乐 | 尺度跳跃 |

### 第1轮回归枢纽动作

| 步骤 | 做什么 |
|------|--------|
| 收集 | 汇总所有守护进程的碰撞产物和回归块 |
| 筛选 | 标记"高价值"种子（confidence=实推 或 value=高 的产物） |
| 排序 | 按潜力排列方向：最有希望成型的排前面 |
| 记录 | 更新轨迹：第1轮发现了什么、踩了什么坑、哪些方向值得追 |
| 种子 | 产出3-5个"思想种子"供第2轮追踪 |

---

## 第2轮：万物萌生（深度追踪）

### 目标
从第1轮的高价值种子中选方向，深挖到底。

### 执行步骤

1. **选种子**：从第1轮的高价值种子中选2-3个最有潜力的
2. **换视角**：每个种子配1-2个新守护进程
   - 可以换一个**完全不同的学科**来看同一个种子
   - 可以用**同一学科换角度**（比如从热力学换成量子力学）
   - 可以尝试**跨种子缝合**（两个种子撞在一起会怎样）
3. **深度碰撞**：守护进程出发，策略更聚焦
4. **收集回归块**

### 守护进程分配示例

| 守护进程 | 追哪个种子 | 用什么学科/角度 | 用什么策略 |
|---------|----------|---------------|----------|
| D6 | 种子1：X理论雏形 | 控制论（新学科） | 结构对撞 |
| D7 | 种子2：Y框架雏形 | 同学科深入 | 逆向复制 |
| D8 | 种子1+2缝合 | 建筑学 | 缺失注入 |

### 第2轮回归枢纽动作

| 步骤 | 做什么 |
|------|--------|
| 合并 | 把第2轮的深度碰撞结果与第1轮的基础合并 |
| 成型 | 识别哪些半成品最接近"成型理论" |
| 补缺 | 检查成型方向是否有逻辑漏洞 |
| 记录 | 更新轨迹：推演路径至此长什么样 |
| 半成品 | 产出1-2个"半成型理论"供第3轮验证 |

---

## 第3轮：三角定圆（反刍验证）

### 目标
用完全不同的角度检验第2轮的半成品，看它站不站得住。

### 执行步骤

1. **选验证学科**：挑前两轮没用过的学科，或最可能"打翻"半成品的学科
2. **压力测试**：
   - 正面攻击：这个理论在你这个学科里说得通吗？
   - 极端场景：把这个理论推到极端，会崩吗？
   - 反例搜证：有没有反例能推翻它？
3. **修整或推翻**：
   - 站住了 → 成型理论
   - 崩了但可修 → 修完再验
   - 崩了不可修 → 标记为"有价值的失败"（为什么错比为什么对更重要）
4. **最终回归**

### 守护进程分配示例

| 守护进程 | 验证什么 | 用什么学科 | 策略 |
|---------|---------|----------|------|
| D9 | 半成品1 | 佛学（前两轮没用过） | 假设对撞 |
| D10 | 半成品1+2 | 天文学（尺度最大） | 尺度跳跃 |

### 第3轮回归枢纽动作

| 步骤 | 做什么 |
|------|--------|
| 终判 | 判定每个半成品：成型 / 可修 / 失败 |
| 闭合 | 对成型理论做最终推演路径整理 |
| 回归笔记 | 写最终回归笔记——让下一个Agent可从枢纽直接启动 |
| 开放问题 | 标记本轮没能解决但值得继续探索的问题 |

---

## 三轮结束后的产出

| 产出 | 说明 |
|------|------|
| **成型理论** | 经过三轮锤炼的新理论/新框架 |
| **推演路径** | 从输入到最终理论，每一步怎么走的 |
| **开放问题** | 值得继续探索但本轮未能解决的 |
| **置信标注** | 每条推理线的置信度（实推/类比/猜想） |
| **回归笔记** | 下一个Agent可从枢纽直接启动 |
| **碰撞JSON** | 供可视化脚本使用的数据 |
