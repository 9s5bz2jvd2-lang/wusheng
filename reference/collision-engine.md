# 碰撞引擎核心机制

> **碰撞不是混合，是化学反应。A+B不是A和B的叠加，而是产生C——一个A和B都不曾有的新东西。**

## 总览

碰撞引擎的工作分为三层：

```
输入内容
    │
    ▼
┌─────────────────────────────┐
│  第1层：深度解析（拆解）      │  把输入拆成底层结构
└─────────────┬───────────────┘
              │
┌─────────────▼───────────────┐
│  第2层：学科映射（匹配）      │  找能和底层结构碰撞的学科
└─────────────┬───────────────┘
              │
┌─────────────▼───────────────┐
│  第3层：碰撞生成（反应）      │  守护进程带策略出去撞
└─────────────────────────────┘
```

## 第1层：深度解析

把输入内容拆解为5种底层结构：

| 底层结构 | 是什么 | 怎么识别 | 例子 |
|---------|--------|---------|------|
| **框架** | 内容用什么结构组织知识 | 分类体系、层次、维度 | 中医辨证=按证型分类；物理学=按力/能量/场分类 |
| **假设** | 内容默认但未明说的前提 | "理所当然"的东西 | 经济学假设"人是理性的"；中医假设"人与自然相应" |
| **边界** | 内容在哪失效、到哪为止 | "例外""不适用于""超出范围" | 牛顿力学在微观失效；营养建议不适用于危重病人 |
| **元模式** | 内容反复出现的思维模式 | 类比、因果、循环、对立统一 | "阴阳"是对立统一；"进化"是因果+随机 |
| **空缺** | 内容该说但没说的 | 逻辑跳跃、避而不谈 | 食养指南没说"合并症怎么吃" |

### 解析输出格式

```yaml
deep_analysis:
  source_summary: "输入内容的一句话概括"
  frameworks: ["框架1", "框架2"]
  assumptions: ["假设1", "假设2"]
  boundaries: ["边界1", "边界2"]
  meta_patterns: ["模式1", "模式2"]
  gaps: ["空缺1", "空缺2"]
```

## 第2层：学科映射

根据解析出的底层结构，匹配学科库中能产生最强烈碰撞的学科：

| 底层结构 | 最能碰撞的学科类型 | 碰撞效果 |
|---------|------------------|---------|
| 框架 | 结构差异大的学科 | 框架裂缝 → 新分析工具 |
| 假设 | 前提相反的学科 | 假设对冲 → 新前提 |
| 边界 | 从边界另一侧出发的学科 | 边界缝合 → 交叉学科 |
| 元模式 | 使用不同模式的学科 | 模式互补 → 新思维方式 |
| 空缺 | 正好在空缺处有知识的学科 | 空缺注入 → 方法迁移 |

### 映射规则

1. 用户指定学科 → 优先使用用户指定的
2. 用户未指定 → 从学科库自动匹配（每个底层结构匹配1-2个学科）
3. 总共派遣3-6个守护进程（太少碰撞不够，太多信息过载）

## 第3层：碰撞生成

每个守护进程携带以下信息出发：

```yaml
daemon_dispatch:
  daemon_id: "D1"
  discipline: "物理学"
  target_structure: "框架"          # 撞哪个底层结构
  strategy: "结构对撞"              # 用哪种策略（参见 six-strategies.md）
  input_snapshot: "..."             # 输入内容的底层结构摘要
```

守护进程碰撞后必须返回：

```yaml
daemon_return:
  daemon_id: "D1"
  discipline: "物理学"
  strategy_used: "结构对撞"
  collision_products:               # 碰撞产物
    - idea: "新想法描述"
      type: "裂缝"                  # 裂缝/缝合/矛盾/迁移/跳跃/逆转
      confidence: "类比"            # 实推/类比/猜想
      value: "高"                   # 高/中/低
      derivation: "从物理学的X框架与输入的Y框架对撞，发现..."
  return_block:                     # 回归块
    reusable_patterns: ["可复用模式1"]
    gotchas: ["踩到的坑"]
    open_questions: ["开放问题"]
    should_promote: ["应提升到枢纽的内容"]
    do_not_promote: ["不应提升的领域特定内容"]
```

## 碰撞产物的6种类型

| 类型 | 是什么 | 怎么产生的 |
|------|--------|-----------|
| **裂缝** | 两个结构之间的断裂带 | 结构对撞发现不重合的地方 |
| **缝合** | 两个领域的衔接点 | 边界对撞发现A的终点=B的起点 |
| **矛盾** | 两个前提的正面冲突 | 假设对撞发现根本对立 |
| **迁移** | 一个领域的方法搬到另一个 | 缺失注入发现刚好补上 |
| **跳跃** | 不同尺度之间的映射 | 尺度跳跃发现宏观规律=微观模式 |
| **逆转** | 模式倒过来套用 | 逆向复制发现"反过来也成立" |

## 一句话

> 拆到底层，找对撞家，带上策略出发——碰撞引擎不搬运知识，它让知识发生反应。
