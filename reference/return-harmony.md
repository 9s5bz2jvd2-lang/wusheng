# 回归调和机制

> **分叉不回归，只是发散；回归不分叉，只是内卷。两者兼备，才是圆。**

## 与圆理论的承接

万物生的回归调和直接继承圆理论的核心架构：

| 圆理论概念 | 万物生对应 | 实现方式 |
|-----------|----------|---------|
| 网以通达 | 第1轮广度碰撞 | 守护进程分叉外出 |
| 球以成身 | 三轮回归积累 | 每轮回归块→枢纽→下一轮更聪明 |
| 心神合一 | 枢纽调和 | 父Agent统一筛选、合并、决策 |
| 我圆如一 | 最终闭合 | 成型理论 + 回归笔记 → 下次可从枢纽启动 |

## 回归块（Return Block）

每个守护进程完成碰撞后，必须返回以下内容：

```yaml
return_block:
  # 碰撞产物
  collision_products:
    - idea: "新想法的简短描述"
      type: "裂缝|缝合|矛盾|迁移|跳跃|逆转"
      confidence: "实推|类比|猜想"
      value: "高|中|低"
      derivation: "这个想法是怎么产生的（2-3句话）"

  # 可复用模式
  reusable_patterns:
    - pattern: "发现的通用模式"
      applicable_to: ["可应用于哪些场景"]

  # 陷阱
  gotchas:
    - gotcha: "踩到的坑"
      lesson: "教训是什么"

  # 开放问题
  open_questions:
    - question: "值得继续探索但本轮无法回答的问题"

  # 提升建议
  should_promote:
    - item: "值得提升到枢纽、供后续轮次和下次碰撞使用的内容"

  # 不提升
  do_not_promote:
    - item: "领域特定、不适合通用化的内容"
```

## 枢纽调和（Hub Harmony）

每轮结束后，父Agent在枢纽执行调和：

### 第1轮调和：筛选

| 动作 | 做什么 | 判断标准 |
|------|--------|---------|
| 收集 | 汇总所有守护进程的回归块 | 无遗漏 |
| 去重 | 合并相同方向的碰撞产物 | 同一idea出现2次以上 = 高信号 |
| 筛选 | 标记高价值种子 | confidence=实推 或 value=高 |
| 排序 | 按潜力排列 | 能否在第2轮深挖成型 |
| 记录 | 更新轨迹 | 第1轮发现了什么、踩了什么坑 |
| 种子 | 产出3-5个思想种子 | 每个种子 = 一个可追踪的方向 |

### 第2轮调和：合并

| 动作 | 做什么 | 判断标准 |
|------|--------|---------|
| 合并 | 第2轮碰撞结果 + 第1轮基础 | 找到可以缝合的线头 |
| 成型 | 识别半成品 | 有完整推理链的 = 半成品 |
| 补缺 | 检查逻辑漏洞 | 推理链断裂处 = 需要第3轮验证的点 |
| 记录 | 更新推演路径 | 从输入到现在走了哪几步 |
| 半成品 | 产出1-2个半成型理论 | 每个半成品 = 有框架但未经验证的理论 |

### 第3轮调和：闭合

| 动作 | 做什么 | 判断标准 |
|------|--------|---------|
| 终判 | 判定每个半成品 | 站住了 = 成型 / 可修 = 修补后再验 / 崩了 = 有价值的失败 |
| 闭合 | 整理完整推演路径 | 每一步都可回溯 |
| 回归笔记 | 写最终回归笔记 | 让下一个Agent可从枢纽直接启动 |
| 开放问题 | 标记未解决问题 | 值得继续但不阻塞本轮闭合 |

## 回归笔记（Return Note）

三轮结束后，枢纽产出最终回归笔记：

```markdown
## 万物生回归笔记

### 输入
[原始输入内容的摘要]

### 碰撞路径
1. 第1轮：[碰撞了哪些学科] → [产出了哪些种子]
2. 第2轮：[追踪了哪些种子] → [产出了哪些半成品]
3. 第3轮：[验证了哪些半成品] → [哪些成型/哪些失败]

### 成型理论
[最终产出的新理论/新框架]

### 推演路径
[从输入到最终理论的完整推理链]

### 可复用模式（提升到枢纽）
- [模式1]：...
- [模式2]：...

### 陷阱（供下次避坑）
- [陷阱1]：...

### 开放问题（值得继续）
- [问题1]：...

### 有价值的失败
- [失败的理论]：为什么它不成立，但为什么尝试它是有价值的

### 不提升的内容（领域特定）
- [内容]：为什么不适合通用化

### 下次入口
[如果继续探索，应从哪里开始]
```

## 不变核心（守一）

| 不变项 | 规则 | 不可违背的原因 |
|--------|------|-------------|
| 可回溯性 | 每个碰撞产物标注来源和路径 | 保证可验证、可追责 |
| 不编造 | 三级置信标注：实推/类比/猜想 | 诚实标注比虚假确定更有价值 |
| 回归纪律 | 守护进程不交回归块 = 该维度不计入调和 | 没有回归的分叉是浪费 |
| 来源标注 | 每条推理线追溯到某个碰撞节点 | 知道新想法从哪来 |
| 版权纪律 | 碰撞产物是原创合成，不是复制粘贴 | 避免侵犯源材料版权 |

> **最终检验：如果下一个Agent能从回归笔记启动，而非从本轮聊天历史启动，这个圆就闭合了。**
