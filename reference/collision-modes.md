# 碰撞模式：向内、向外、无限

> **万物生的碰撞有三种模式。三种模式可以叠加——内部碰撞发现问题，外部碰撞寻找解法，无限碰撞推演未来。**

## 总览

| 模式 | 一句话 | 碰撞方向 | 用户限定 |
|------|--------|---------|---------|
| **内部碰撞** | 向里挖，在自己的结构里找裂缝 | 输入内容自身组件之间互相碰撞 | 限定输入范围 |
| **外部碰撞** | 往外撞，拉别的学科进来对轰 | 输入内容与其他学科碰撞 | 可选指定学科 |
| **无限碰撞** | 撒开了跑，哪里都撞一下看看 | 无边界，动态发现一切可碰撞的对象 | 不做限定 |

三种模式的选择权在**用户手中**。万物生不替用户决定该往哪撞，只提供碰撞的**方向框架**。

---

## 模式一：内部碰撞（向内求索）

### 什么时候用

- 用户有一段代码，想在**同一套逻辑框架**内找优化空间
- 用户有一份资料，想找**内部矛盾**和不自洽的地方
- 用户有一个理论，想推演它的**隐含推论**和自我局限性
- 用户有一组数据，想发现**隐藏的模式**和异常点

### 怎么做

不引入外部学科。把输入内容自身拆解为组件，让组件之间互相碰撞。

```
输入内容
    │
    ▼
深度解析 → 拆为多个组件（模块、章节、数据段、概念节点）
    │
    ├──→ 组件A vs 组件B → 有没有矛盾？有没有重复？有没有空白？
    ├──→ 组件A vs 组件C → ...
    ├──→ 理论前提 vs 实际数据 → 有没有被打脸的地方？
    └──→ 显性结构 vs 隐性结构 → 说出来的和实际做的有什么不同？
              │
              ▼
    回归枢纽 → 筛选有价值发现 → 第2轮深度 → 第3轮验证
```

### 内部碰撞的策略

| 策略 | 怎么做 | 产出 |
|------|--------|------|
| **自反碰撞** | 输入内容的某个断言 vs 它的反面 | 发现隐含的假设和边界 |
| **间隙碰撞** | 组件A 的终点 vs 组件B 的起点 | 发现缺失的过渡和未覆盖的区域 |
| **递归碰撞** | 上一轮碰撞的产物 vs 原始输入 | 发现元层面的规律 |
| **镜像碰撞** | 输入内容 vs 自身倒过来的版本 | 发现非对称性和隐藏偏向 |
| **密度碰撞** | 高频出现的概念 vs 低频出现的概念 | 发现"被忽视的部分" |
| **时间碰撞** | 内容的早期部分 vs 晚期部分 | 发现一致性漂移和演化路径 |

### 例子

| 输入 | 内部碰撞 | 可能产出 |
|------|---------|---------|
| 一段200行代码 | 模块拆分→模块间自反碰撞 | "这段代码有3处重复逻辑可以抽象""这个函数名字和它实际做的事不匹配" |
| 一篇论文 | 论点分解→论点间间隙碰撞 | "论证链在第3步断了""前面的假设和后面的结论之间存在隐含跳跃" |
| 一份食养指南 | 证型间镜像碰撞 | "3个证型的建议互相矛盾""所有证型对齐后发现有7种食材从未被推荐" |

## 模式二：外部碰撞（向外拓展）

### 什么时候用

- 用户有一个领域内的知识，想拉**其他领域**来碰撞
- 用户想验证一个理论在**其他学科**的框架下是否成立
- 用户想从**跨学科**的角度寻找新解法

### 怎么做

这是万物生的**核心模式**，即现有的三轮碰撞流程（参见 `three-rounds.md`）。

**用户限定**：
- 用户可以指定碰撞学科（如"用天文学、道家、舞蹈碰撞"）
- 用户可以指定学科类别（如"只用自然科学的学科碰撞"）
- 用户可以不指定，让系统从学科库动态匹配（参见 `discipline-library.md`）

**碰撞策略**：六种对撞策略全适用（参见 `six-strategies.md`）

### 用户限定的意义

外部碰撞的威力取决于**碰撞距离**——两个领域离得越远，撞出来的东西越新颖，但也越可能"不靠谱"。用户限定学科范围 = 控制碰撞距离。

| 限定方式 | 碰撞距离 | 产出特点 |
|---------|---------|---------|
| 指定同类学科 | 近距 | 修补型创新（裂缝/缝合），实用但不够颠覆 |
| 指定异类学科 | 中距 | 迁移型创新（跳跃/注入），有惊喜但需验证 |
| 不限 | 远距→极远 | 革命型创新（逆转/矛盾），可能颠覆也可能胡说 |
| 指定完全无关学科 | 极远 | 最不可预测，但也是最能"生出新物种"的方向 |

## 模式三：无限碰撞（任其生灭）

### 什么时候用

- 用户说"**随便碰，想看能生出什么**"
- 用户不确定该限定在什么范围
- 用户追求**绝对的新颖性**，不在乎每一条都能落地

### 怎么做

不设任何边界。输入内容解析后，不做学科匹配的限定，让系统**动态发现一切可碰撞的对象**——包括预设学科库里的、库外的、甚至不是学科的东西（一句话、一首诗、一张图片描述、一个随机概念）。

```
输入内容
    │
    ▼
深度解析（但更开放：连"可以跟什么碰"都交给系统去发现）
    │
    ├──→ 守护进程A：找一个最近的学科 → 碰撞
    ├──→ 守护进程B：找一个最远的学科 → 碰撞
    ├──→ 守护进程C：不是学科，是一个随机概念 → 碰撞
    ├──→ 守护进程D：不是概念，是一个问题 → 碰撞
    └──→ 守护进程E：上一轮的产物 → 随机转个方向再撞
```

### 无限碰撞的独特策略（除六种常规策略外）

| 策略 | 怎么做 |
|------|--------|
| **随机跳跃** | 从学科库（甚至库外）随机选一个对象碰撞，不筛选 |
| **反向推理** | 先生成一个结论，再反推"这个结论可能从什么碰撞中产生" |
| **空白创造** | 假定输入内容**完全错误**，然后从错误中推论出一个新框架 |
| **极端化** | 把输入的某个特征推到极端 → 观察崩溃点 → 崩溃点附近有新东西 |
| **空白注入** | 故意留白——放弃输入的某部分结构，看看剩下来的东西怎么自组织 |

### 无限碰撞的"危险"

无限碰撞产出的理论中，可能有大量"不靠谱"的——置信度标注为「猜想」的会占多数。但这正是无限碰撞的目的：**不是每一枪都命中，但只要中一枪，就是别人从来没打过的方向。**

无限碰撞的产物需要用户自己筛选——万物生只负责生，用户负责裁决。

## 三种模式的组合使用

三种模式不是互斥的，可以叠加：

| 叠加方式 | 场景 |
|---------|------|
| **先内部、后外部** | 先用内部碰发现问题和裂缝 → 再用外部碰撞找其他领域的解法 |
| **先外部、后内部** | 先拉多个学科碰撞 → 对产出的理论做内部碰撞（自反验证） |
| **外部+无限叠加** | 指定核心学科做外部碰撞 → 同时做无限碰撞补充意外发现 |
| **三轮模式切换** | 第1轮用内部（找到裂缝）→ 第2轮用外部（找解法）→ 第3轮用无限（推演未来） |

---

## 用户限定语法

万物生从用户输入中识别碰撞模式：

| 用户说 | 识别为 |
|--------|--------|
| "用XX学科碰撞..." | 外部碰撞，指定学科 |
| "只限定在XX碰撞" / "内部碰撞" | 内部碰撞 |
| "随便碰" / "不限定" / "看看能生出什么" | 无限碰撞 |
| "先内部再外部..." | 组合模式，按顺序执行 |
| 只说输入内容，没提限定 | 默认外部碰撞，学科库动态匹配 |

> **万物生不替用户决定方向。用户是船长，万物生是引擎——告诉船往哪开，引擎才会全力发动。**
