# 碰撞指南编写法

> **杠杆在指南里，不在模型里。**
> The leverage is in the guide, not the model.

## 来源

本方法论受 Bun/Claude Code 的 `PORTING.md` 启发——一个 30KB 的翻译指南，让 Claude Code 在 11 天内将 100 万行 Zig 代码翻译为 Rust，通过率 99.8%。

**核心洞察**：不是"A I很聪明所以干得快"，是"人类写了一份极好的指南，AI 照着干所以干得快"。

万物生将这一方法论**逆向迁移**到跨学科创造力场景中——写一份好的碰撞指南，能让万物生的三轮碰撞精准度大幅提升。

---

## 核心原则

### 1. 人类定边界，AI 在边界内跑

> PORTING.md: "预先锁定每一个的类型映射，不让 AI 临时判断。"

无效的指南：**"帮我想想食养助手还能怎么改进。"**
有效的指南：**"用生态学的食物链概念碰撞食养助手的证型分类框架，只碰结构层面，不碰内容细节。碰撞策略用结构对撞。"**

| 维度 | 无效指南 | 有效指南 |
|------|---------|---------|
| **范围** | "随便碰" | "只碰分类框架，不碰具体食材" |
| **策略** | "用你最好的判断" | "用结构对撞和假设对撞两种策略" |
| **产出** | "有新发现就好" | "产出至少1个"合并症方案"的雏形，标注置信度" |
| **学科** | "可以碰任何学科" | "优先碰哲学、经济学、天文学，不要碰医学" |

### 2. 分阶段，别一次全做

> PORTING.md: "一次全做，AI 的上下文会崩塌。"

万物生已有三轮碰撞的分阶段设计——这本身就是"分阶段"的体现。但碰撞指南可以**在每一轮开始前细化阶段目标**：

| 阶段 | 指南应该告诉 AI 的 | 不应该让 AI 自己决定的 |
|------|-------------------|---------------------|
| 深度解析 | 优先提取哪类底层结构（框架？假设？元模式？） | 怎么提取 |
| 第1轮 | 最多碰撞几个学科 | 选哪些学科 |
| 第2轮 | 从哪些种子出发 | 怎么深挖 |
| 第3轮 | 用什么角度验证 | 怎么判断通过or失败 |

### 3. 不确定就打标签，别猜

> PORTING.md: "// TODO(port): <原因> — 标记优于猜测。"

在万物生中，碰撞指南可以预设"不确定地带"——告诉 AI，某些方向上**别瞎猜**，直接标记"此处需要人类判断"：

```yaml
collision_guide:
  known: ["框架可碰", "假设可碰"]
  uncertain: ["边界可碰但需要人类验证", "元模式暂时不碰"]
  do_not_touch: ["具体食谱数据不要碰"]
```

三个标签：
- ✅ `known`: 放心碰
- ⚠️ `uncertain`: 可以碰但产出的置信度上限为"类比"，碰完需要人类验证
- 🚫 `do_not_touch`: 绝对不碰

### 4. 禁止清单比允许清单更重要

> PORTING.md: "禁止使用 tokio、rayon、async fn……"

万物生的碰撞指南里，**告诉 AI 不该做什么，比告诉它该做什么更关键**：

```yaml
collision_guide:
  forbidden:
    - 不碰宗教信仰敏感的领域
    - 不产出"取代医生"类建议
    - 不编造统计数据
    - 不在第1轮就产出"成型理论"（那是第3轮的事）
    - 不对用户输入做"好坏"评价
```

## 碰撞指南模板

### 精简版（日常使用）

```yaml
碰撞指南:
  输入: "食养助手系列的证型分类框架"
  模式: 外部碰撞
  学科: ["生态学", "网络科学", "博弈论"]
  不碰: ["宗教学", "具体食谱数据"]
  策略: ["结构对撞", "假设对撞", "边界缝合"]
  阶段:
    第1轮目标: "找出至少3个框架裂缝，每个裂缝产出1个新分类思路"
    第2轮目标: "从裂缝中选择2个最有潜力的，深度推演为新框架雏形"
    第3轮目标: "用控制论和热力学对第2轮产物做压力测试"
  uncertain: ["置信度为'猜想'的产物需要人类二次判断"]
  forbidden:
    - "不碰具体食谱数据"
    - "不产出超越食养能力范围的医疗建议"
    - "不编造统计数据"
```

### 完整版（重度使用）

```yaml
碰撞指南:
  # === 基本信息 ===
  输入: "完整描述输入内容"
  输入类型: "代码 | 文本 | 数据 | 理论 | 问题 | 混合"
  模式: "内部碰撞 | 外部碰撞 | 无限碰撞 | 组合模式（写明顺序）"

  # === 碰撞范围 ===
  学科:
    include: ["指定学科1", "指定学科2"]
    exclude: ["排除学科1"]
    category: "自然科学 | 社会科学 | 人文学科 | 艺术 | 工程 | 东方智慧 | 不限"

  策略:
    include: ["结构对撞", "假设对撞"]
    exclude: ["逆向复制"]

  底层结构:
    focus: ["框架", "假设"]      # 优先提取的结构
    skip: ["空缺"]                 # 跳过不碰的结构

  # === 阶段控制 ===
  阶段目标:
    第1轮: "具体可衡量的目标"
    第2轮: "具体可衡量的目标"
    第3轮: "具体可衡量的目标"

  # === 安全边界 ===
  uncertain:
    - "领域1: 可碰但置信上限为'类比'"
    - "领域2: 碰完需要人类二次验证"

  forbidden:
    - "不碰XXX"
    - "不产出XXX类建议"
    - "不编造XXX"

  # === 产出要求 ===
  产出:
    最低数量: 2               # 最少产出几个成型理论
    置信度门槛: "类比"         # 低于此置信度的不纳入最终产出
    可视化: true               # 是否生成可视化
    回归笔记: true             # 是否生成回归笔记

  # === 深度偏好 ===
  碰撞距离: "远距"            # 近距 | 中距 | 远距 | 极远
  创新偏好: "革命型"          # 修补型 | 迁移型 | 革命型
  反刍强度: "强"              # 第3轮压力测试的强度
```

## 编写指南的最佳实践

### 实践1：先做一次"裸奔碰撞"，再写指南

不要一开始就写指南。先让万物生**裸奔一次**（不给任何额外限定，只给输入内容），看看它撞出了什么。然后从碰撞结果中找：

- **哪些碰撞是浪费的** → 写进 `forbidden`
- **哪些方向值得深追** → 写进 `focus`
- **哪些策略最有效** → 写进 `include`

裸奔一次 = 花 1 次低质量碰撞，换来 N 次高质量碰撞。

### 实践2：指南本身也在迭代

指南不是一锤子买卖。每次碰撞结束后，回归笔记里会记录"这次踩了什么坑""这次发现了什么可复用的模式"——把这些写进**下一版指南**。

```
第N次碰撞的回归笔记
    │
    ▼
更新碰撞指南（加入新发现的 forbidden/uncertain/focus）
    │
    ▼
第N+1次碰撞（更精准）
```

这本身就是圆理论的体现——碰撞的产物（回归笔记）回归到指南的编写过程中，指南越来越好，碰撞越来越精准。

### 实践3：指南不是束缚，是瞄准镜

指南不是限制创造力，而是**让创造力集中在有价值的方向上**。没有指南的碰撞 = 散弹枪打兔子，有指南的碰撞 = 狙击枪打兔子。

## 一图总结

```
┌─────────────────────────────────────────────────┐
│              碰撞指南 = 瞄准镜                       │
├───────────────┬─────────────────────────────────┤
│ include       │ 优先打的方向                        │
│ exclude       │ 不要打的方向                        │
│ forbidden     │ 绝对不碰的红线                       │
│ uncertain     │ 打了之后需要人确认                     │
│ focus         │ 对准哪个底层结构                      │
│ 阶段目标       │ 每一轮要打出什么结果                   │
├───────────────┴─────────────────────────────────┤
│  写在指南里的 vs 留给AI判断的                       │
│  人类定边界 → 万物生在边界内自由奔跑                   │
└─────────────────────────────────────────────────┘
```

> **好的指南是杠杆——它把人类的判断力放大 N 倍，同时保留 AI 的创造力。**
