# 学科库管理

> **学科是碰撞的弹药库。用户可以自带弹药，也可以让系统自动配弹。**

## 设计原则

- **用户主权**：学科由用户决定，系统只建议不强制
- **预设+动态**：有默认库打底，也支持动态发现和自定义
- **跨域优先**：尽量选和输入领域差异大的学科，差异越大碰撞越强烈

## 动态发现

当用户未指定学科时，系统根据输入内容的底层结构自动匹配：

### 匹配逻辑

| 输入的底层结构 | 优先匹配的学科大类 | 匹配规则 |
|-------------|------------------|---------|
| 框架清晰 | 与框架差异大的大类 | 结构对撞最有冲击 |
| 假设明显 | 与假设对立的大类 | 假设对撞最能生新知 |
| 边界明确 | 从边界另一侧出发的大类 | 边界缝合最自然 |
| 元模式强 | 使用不同模式的大类 | 模式互补最丰富 |
| 空缺显著 | 空缺处有知识的大类 | 注入迁移最实用 |

### 匹配数量

- 第1轮：3-6个学科（广度优先）
- 第2轮：2-3个学科（深度优先，可能含新学科）
- 第3轮：1-2个学科（验证优先，选前两轮没用过的）

## 自定义学科

用户可随时添加不在预设库中的学科：

```yaml
custom_discipline:
  name: "用户定义的学科名"
  core_thinking: "这个学科的核心思维方式"
  good_for: ["适合碰撞的底层结构1", "底层结构2"]
  example_insight: "这个学科的一个代表性洞见"
```

> **万物生不限制学科范围。万物皆可碰撞。**
